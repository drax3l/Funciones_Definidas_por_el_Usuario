Taller 3.3 · Funciones Definidas por el Usuario (UDF)
Base de datos: Northwind – SQL Server

Introducción: ¿Qué son las UDFs?

Las Funciones Definidas por el Usuario (UDFs) en SQL Server son bloques de código T-SQL reutilizables que reciben parámetros y retornan un resultado (un valor o una tabla).

¿Por qué son importantes?
  • Permiten modularizar la lógica de negocio.
  • Simplifican consultas complejas y las hacen más legibles.
  • Facilitan la reutilización en múltiples consultas o reportes.
  • Ayudan a mantener y estandarizar reglas de negocio dentro de la base de datos.

En este taller desarrollamos 10 UDFs sobre la base de datos Northwind, incluyendo funciones escalares, de tabla inline y de tabla multi–sentencia.

Objetivos del Taller

  • Diseñar e implementar funciones UDF en SQL Server usando T-SQL.
  • Diferenciar entre funciones escalares y funciones de tabla.
  • Aplicar UDFs para resolver problemas reales en la base de datos Northwind.
  • Preparar un script .sql único con las 10 funciones y sus pruebas.
  • Documentar la solución mediante un informe técnico y evidencias de ejecución.
Este póster resume los puntos clave que presentamos en el informe: introducción, arquitectura, descripción de funciones y aprendizajes.

Tipos de Funciones Implementadas

Funciones Escalares

Retornan un único valor (texto, número, fecha).
Ejemplos: cálculos, validaciones, formateo.

Funciones de Tabla (Inline)

Retornan una tabla derivada basada en una sola consulta SELECT.

Funciones de Tabla (Multi)

Construyen una tabla paso a paso usando varias sentencias dentro de la función.

Cada función cuenta con un SELECT de prueba para validar su funcionamiento en SQL Server.

Ejemplos de Código para la Expo

Ejemplo de función escalar: CalcularTotalPedido

CREATE OR ALTER FUNCTION CalcularTotalPedido (@OrderID INT)
RETURNS DECIMAL(10,2)
AS 
BEGIN
    DECLARE @Total DECIMAL(10,2);
    
    SELECT @Total = SUM(UnitPrice * Quantity * (1 - Discount))
    FROM [Order Details]
    WHERE OrderID = @OrderID;
    
    RETURN ISNULL(@Total, 0);
END;
-- Prueba:  SELECT dbo.CalcularTotalPedido(10248) AS TotalPedido;

Ejemplo de función de tabla (inline): ProductosPorCategoria

CREATE OR ALTER FUNCTION ProductosPorCategoria (@CategoryID INT)
RETURNS TABLE
AS
RETURN
(
    SELECT 
        p.ProductID,
        p.ProductName,
        p.UnitPrice,
        p.UnitsInStock,
        c.CategoryName
    FROM Products p
    JOIN Categories c ON p.CategoryID = c.CategoryID
    WHERE p.CategoryID = @CategoryID
);
-- Prueba:  SELECT * FROM dbo.ProductosPorCategoria(1);

Arquitectura de Conexión

El taller se planteó para trabajar con tres bases de datos en diferentes entornos:

  • Northwind · Instancia local de SQL Server (funciones UDF del taller).
  • Pubs · Base de datos en un servidor de datacenter (conexión remota).
  • AdventureWorks · Base de datos en la nube (ej. Azure / somee.com).
Para este entregable, nos centramos en las funciones sobre Northwind, pero la arquitectura permite extender la lógica a otras bases con la misma idea de UDFs.

Conclusiones y Aprendizajes

  • Consolidamos el uso de T-SQL para crear funciones reutilizables.
  • Comprendimos cuándo conviene usar funciones escalares y cuándo funciones de tabla.
  • Vimos que una buena documentación (comentarios, ejemplos de uso) facilita la lectura del código.
  • Comprobamos la importancia de realizar pruebas unitarias con SELECT específicos por función.
  • Identificamos cómo las UDFs ayudan a centralizar la lógica de negocio dentro de la base de datos.
Estas ideas se desarrollan con más detalle en el Informe Técnico (.pdf) y en el script .sql con las 10 funciones y sus pruebas.
